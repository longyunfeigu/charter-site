Charter 源于一个朴素的观察：最强的 coding agent 活在命令行里， 而体验最好的 IDE 都内置自家的 agent——并要你将就。 你不该被迫二选一：一边是你信任的 agent，一边是你应得的驾驶舱。

看得见的 Agent 工作 · 从 Prompt 到 Proof

  • Agent CLI

    前沿，在终端里

    Claude Code 和 Codex 代表当今最强。但终端给你的只是滚动的输出， 不是掌控——没有 Diff 界面、没有审查门、没有回放。

  • Charter

    它们的 IDE，你的规则

    给你已经信任的 agent 一个 IDE 外壳——编辑器、Worktree、真终端， 再加上 CLI 从未有过的证据、门与回滚。同时保留终端之魂： 持久 PTY、可读的命令块、多路复用的 Session。

  • AI-first IDE

    漂亮的壳，别人的驾驶员

    驾驶舱很精致，但 agent 是内置的、是他们的。 编辑器归你；键盘上的那双手，替你选好了。

Session = project+ agent+ worktree+ conversation+ plan+ files+ terminals+ preview+ verification+ review

Session 是一次完整人机协作的基本单元。关于这个任务的一切，都不在别处。

一次委托，三个时刻

下面每一屏都是真实应用在处理同一个 Session： 选好帮手，看着过程，裁决结果。

01 · 委托

一个 Composer，装下每个 agent

启动内置的 Charter Agent，或原生运行你已安装的 Claude Code 与 Codex CLI——同一套 Session 模型、同一道审查门，不用切换产品外壳。

每个任务给多长的缰绳，由你定：Read、 Approve、Auto 或 Full——从只读问答到受监督的自主。

02 · 观察

证据就在对话旁边

Agent 运行时，文件写入、命令、Diff、预览、检查与决策组成同一条 按时间排列的账本——带时间戳和增删统计，实时更新。

不用在聊天记录里考古。工作日志就是界面本身， 结束后的 Session 还能逐步回放。

03 · 裁决

先见证据，再谈批准

Agent 说“完成”只意味着进入待审查。审查界面由真实记录的 证据构成——改动文件、验证历史、最终结果——而不是模型自己的一段总结。

查看带行号的 Diff，运行记录在案的检查，然后请求修改、回滚， 或者批准——整单、按文件，或按代码块。

可以撤销的自主

多数工具只让模型“更容易启动”。Charter 关心的是你按下回车之后的一切—— 在 prompt 和你的代码库之间立着四道门，每一道都留下证据。

  1. 01

    计划

    动笔之前，agent 先提出步骤、影响范围和验证方式。 你可以批准、修改或驳回。

  2. 02

    权限

    每次工具调用都经过同一个网关做策略检查。危险操作先询问； 禁止的操作根本不会执行。

  3. 03

    验证

    记录在案的检查在真实 Worktree 里运行。失败会留在记录上—— 重跑永远不会抹掉历史。

  4. 04

    审查

    完成只会停在待审查，永不自动接受。回滚是字节级的—— 即使你已经接受了也还能回。

即使是需要用户主动选择的 Full 自主模式，也保留同一个网关、 审计痕迹与硬性禁区——验证失败会退回审查，已接受的工作依然可以回滚。

权限是渐变，不是开关

操作按后果分级。最危险的那一端不是弹一个警告框——而是根本不给运行。

R0 读取 读文件、搜索、诊断、git status 允许
R1 工作区写入 在隔离 Worktree 内创建或编辑文件 询问，或按计划与模式放行
R2 本地执行 已知的本地命令与验证检查 已知检查可运行；未知命令询问
R3 外部 / 不可逆 联网或有后果的操作 每次都需显式确认
R4 禁止 sudo、git push、读取密钥、越出工作区的写入 产品直接拒绝

R4 没有“仍然运行”按钮。

模型永远碰不到你的文件

模型回路运行在隔离进程中，没有文件系统、shell 或密钥访问权。 每个工具请求都要跨过同一个受审计的网关——校验格式、检查边界、 执行策略，并把证据记录在案。

  • 用架构隔离

    界面、沙箱化桥接、桌面主进程与模型进程是彼此独立的信任边界。 工具执行发生在墙的主进程一侧，永远不在模型进程里。

  • 默认可逆

    改动发生前先做快照，写入是原子的、带版本校验。 回滚按字节精确恢复，并逐一验证哈希。

  • 密钥待在钥匙串里

    Provider 凭据由操作系统级加密存储，永远不会进入界面—— 界面只知道密钥存在。

  • 诚实的本地优先

    项目、任务状态与证据都留在你的机器上。Prompt 和你附加的上下文 只发往你自己配置的模型端点——并遵循该 Provider 的数据政策。

应用层权限不等于操作系统级沙箱。批准前请审阅命令； 处理不受信任的仓库时请使用隔离环境。

底下是一个完整的 IDE

门是故事的主线——但日常工具一件不缺。 Charter 是一个能干活的 IDE，不是套着文件树的聊天窗口。

  • 完成通知

    任何 Session 结束时，一条带色的通知滑入——成功绿、待审琥珀、 失败红——对应的 Session 会在侧栏持续脉动，直到你打开它。

  • 实时在场

    Agent 写入时文件卡片泛起涟漪，呼吸灯精确指出工作正发生在哪里—— 就在它发生的那一刻。

  • 真终端

    持久 PTY 加上能读懂命令的块模型——长命令跑完了会告诉你。 切换 Session 时 PTY 保持存活。

  • Session 回放

    把结束的 Session 一步步拖着看，看每次改动如何落地—— 只读，不碰你的 Worktree。

  • 技能库

    为可复用的工作流准备的受管技能库，在 Composer 里 输入 / 直接唤出。

  • 随你调校

    四套联动皮肤、明暗双色、命令面板、快速启动器—— 以及默认关闭的数据统计。

  • 真正的编辑器

    Monaco：分屏、查找替换、大文件优雅降级。 Agent 在你身边编辑时，你的光标稳如泰山。

  • 内置 Git

    状态、Diff、暂存、提交、分支。而 git push 永远不是 agent 的工具——这是设计。

  • 全项目搜索

    ripgrep 级速度的全仓搜索，替换在碰到任何文件之前 先给你预览。

  • 代码智能

    TypeScript 与 JavaScript 的完整语言支持，Python 基础支持—— agent 看到的诊断和你是同一份。

  • 实时预览

    看着应用在它的 Worktree 里运行、内嵌展示——然后直接在画面上 画框圈元素，把视觉反馈发给 agent。

  • 飞行中转向

    给运行中的 agent 改方向、排后续任务，或者立即停止—— 而且停止永远不会被粉饰成成功。

今天就从源码构建

Charter 是开发预览版：桌面工作流已实现并通过测试， 安装包、签名与自动更新仍在进行中。从源码构建是当前受支持的使用方式。

terminal
# Node.js ≥ 22.19、npm 和 Git
$ git clone https://github.com/longyunfeigu/Charter.git
$ cd Charter && npm install
$ npm run dev
  1. 打开一个 Git 项目。Charter 面向真实仓库工作，在隔离的 Worktree 里进行。
  2. 添加模型 Provider。设置 → Models：Anthropic、OpenAI、OpenRouter、LiteLLM 或任意兼容端点。密钥存进系统钥匙串。
  3. 创建一个 Session。选择 agent 与权限模式，描述你想要的结果——等它进入待审查时，读证据说话。

想要安装包？在 GitHub 关注 Releases——打包发布正是当前里程碑。

问题，直说直答

为什么还没有安装包？

刻意的排序。让 Charter 值得一用的那部分——隔离改动、证据、验证、审查、回滚—— 先做完并打磨扎实；打包、签名和更新通道是正在进行的里程碑。今天最诚实的 提供方式就是 git clone，所以这个页面上写的就是它。

我的代码会去哪里？

默认哪儿也不去。项目、任务状态和证据账本都在你的机器上。Agent 运行时， 你的 prompt 和附加的上下文只发往你自己配置的模型端点——中间没有别的后端， 数据统计也默认关闭。

能用哪些模型和 agent？

内置的 Charter Agent 支持 Anthropic、OpenAI、OpenRouter、LiteLLM 预设， 以及自定义的 Anthropic 或 OpenAI 兼容端点。已安装的 Claude Code 和 Codex CLI 则原生运行在同一套 Session 与审查模型中。

和 AI-first IDE 有什么不同？

Charter 不和你的 agent 竞争——它和你 agent 蜗居的那个终端窗口竞争。 AI-first IDE 内置它们自家的 agent；Charter 则把你已经在用的 agent 包进 隔离 Worktree、记录账本、可重跑的验证、默认人工批准的审查门， 以及接受之后依然有效的回滚。

真的开源吗？

MIT 协议，公开开发——包括产品规格说明、架构决策记录和实现状态。 如果这个页面上有任何说法在仓库里找不到依据，欢迎提 issue。

Charter 是谁在开发？

主要是 coding agent——就按这个页面描述的方式工作：对着公开的规格说明， 每次改动都有计划、验证与审查。这些门之所以存在，正是因为 Charter 自己的 开发就依赖它们。
